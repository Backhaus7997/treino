## Post-verify fixes (2026-05-12)

User ran TASK-009 smoke run and caught a visual bug not covered by tests:

- **Play icon bug**: `EmpezarEntrenamientoCard._ctaLabel` had `'▶ EMPEZAR ENTRENAMIENTO'` (Unicode `▶` U+25B6 baked into label). On Windows, the OS renders that character with Segoe UI Emoji (colored emoji font), producing an orange/red square instead of a clean play triangle. The `HomeCTAButton` already had a `leadingIcon` param designed for this — the call site just wasn't using it.

Fixes applied directly by orchestrator (not via sub-agent):

1. `lib/features/home/widgets/empezar_entrenamiento_card.dart`:
   - `_ctaLabel` → `'EMPEZAR ENTRENAMIENTO'` (no `▶`)
   - `HomeCTAButton(...)` invocation now includes `leadingIcon: TreinoIcon.play`
2. `test/features/home/widgets/empezar_entrenamiento_card_test.dart`:
   - REQ-HOME-EMPEZAR-001 test updated to assert `label == 'EMPEZAR ENTRENAMIENTO'` AND `leadingIcon == TreinoIcon.play`

After this fix, smoke run visually matched mockup. **TASK-009 → DONE**.

Verify phase (after the fix) found 1 CRITICAL (doc-only — spec/tasks/design still referenced the old `▶` label) + 2 WARNINGs (missing test scenarios). Resolved:

3. **CRIT-001** (doc sync) — `spec.md` lines 119 + 183, `tasks.md` TASK-004a done-when + TASK-004b notes, `design.md` §3 widget tree → all updated to drop `▶` and reference `leadingIcon: TreinoIcon.play`.
